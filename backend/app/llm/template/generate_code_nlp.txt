## Step 2 - Ask what is specific about THIS text dataset

Before writing code, answer these questions mentally:

- What domain is this? (reviews, support tickets, survey responses, news, social media…)
  Let the domain decide what "interesting" means:
  - Reviews → sentiment by rating/category, what drives negative sentiment, topic clusters per star level
  - Support tickets → which topics have the longest/angriest texts, resolution patterns
  - Survey responses → what do unhappy respondents say that happy ones don't?
  - News/articles → topic drift over time, length vs engagement

- Does the dataset have non-text columns (ratings, dates, categories, counts)?
  If yes, the most interesting analyses are CROSS-ANALYSES: sentiment × rating,
  topic × category, text length × outcome. Pure text-only analyses are less valuable.

- What would a domain expert find surprising?
  - A topic cluster that is unexpectedly negative
  - Short texts that have extreme sentiment (angry one-liners vs thoughtful long reviews)
  - A keyword that appears in both positive and negative contexts

## Step 3 - Write code that reflects what you found

### What to compute

GOOD computations - these reveal structure beyond the pre-computed features:
  - Sentiment scores joined with a non-text column: avg sentiment per rating tier, per category
  - Texts at sentiment extremes: top 5 most negative AND most positive, with their metadata
  - Cross-tab: which topic cluster has the highest negative sentiment %?
  - Length × sentiment: do very_long texts have different sentiment than short texts?
  - Keyword presence: for a domain-relevant keyword, do texts containing it skew more negative?

BAD computations (do not use these as standalone keys):
  - Recomputing sentiment stats already in nlp_features  ← already provided
  - Listing top keywords verbatim from nlp_features without new analysis  ← no added value
  - Generic describe() on text length  ← use the length_distribution already provided

### How to name keys

Keys must describe what the value reveals:
  GOOD: "negative_cluster_breakdown", "sentiment_by_rating_tier", "extreme_negative_texts"
  BAD:  "sentiment", "keywords", "summary"

CORRECT USAGE EXAMPLES:
```python
# Cross-analysis: sentiment by non-text column
scores = nlp_features["review_text"]["sentiment"]["scores"]
df2 = df.copy()
df2["sentiment_score"] = scores
result["sentiment_by_rating"] = df2.groupby("rating")["sentiment_score"].mean()

# Extreme texts
result["top_negative"] = df2.nsmallest(5, "sentiment_score")[["review_text", "rating", "sentiment_score"]]

# Topic cluster sentiment
cluster_sentiment = {{}}
for c in nlp_features["review_text"]["topic_clusters"]:
    kw = c["keywords"][0]
    mask = df["review_text"].str.lower().str.contains(kw, na=False)
    avg = df2.loc[mask, "sentiment_score"].mean()
    cluster_sentiment[c["topic"]] = round(avg, 3)
result["cluster_sentiment"] = cluster_sentiment
```
