## Step 3 - Write code that reflects what you found

### What to compute

Choose computations that reveal structure the stats summary cannot show alone:

  GOOD computations:
  - Salary IQR ratio (p75/p25) per country - shows inequality, not just average
  - Roles where ai_risk_score > 0.6 AND salary < median - the "high risk, underpaid" segment
  - Pareto check: what % of job_openings come from top 3 job titles?
  - Cross-tab: survival_class distribution within each education level
  - Conditional correlation: does salary↔risk hold within each country separately?

  BAD computations (do not use these as standalone keys):
  - df.describe()               ← redundant with the stats already provided
  - df["col"].value_counts().head(5)  ← already visible in stats; only use if you filter first
  - df[cols].corr()             ← full matrix already noted in stats; pick a specific pair instead
  - mean ± 2*std outliers on a skewed column  ← use IQR when stats say "right-skewed"

### How to name keys

Keys must describe what the value reveals, not what operation produced it:

  GOOD: "high_risk_underpaid_roles", "salary_inequality_by_country", "pareto_top3_job_concentration"
  BAD:  "summary", "correlation", "outliers", "top_5_jobs"

### Outlier detection - match the method to the distribution

  Skewed column (stats says right-skewed or strongly right-skewed):
    q1, q3 = df["col"].quantile([0.25, 0.75])
    iqr = q3 - q1
    outliers = df[(df["col"] < q1 - 1.5*iqr) | (df["col"] > q3 + 1.5*iqr)]

  Symmetric column (stats says roughly symmetric):
    mu, sigma = df["col"].mean(), df["col"].std()
    outliers = df[(df["col"] < mu - 2.5*sigma) | (df["col"] > mu + 2.5*sigma)]

## Chart instructions

Include a `_charts` key with up to 3 charts.

Rules:
- Each chart must answer a DIFFERENT question - not the same metric in a different format
- If a finding is better shown as a table in `result`, skip the chart for it
- Charts should complement the result keys, not duplicate them

Available types:
  "bar"         → rankings, group comparisons (≤ 15 categories)
  "line"        → change over time ONLY; labels must be time-ordered date strings
  "pie"         → part-of-whole for exactly 2–5 categories; use sparingly
  "scatter"     → relationship between two numeric columns
  "histogram"   → distribution shape of one numeric column
  "grouped_bar" → 2–3 metrics side-by-side per category; requires series_labels

Format rules:
- Every chart: type, title (< 10 words), labels (list[str]), data (list[number] or list[list])
- grouped_bar: also needs series_labels (list[str])
- scatter: also needs data=[[x,y],...] and color_by (see below)
- labels and data must have the same length
- All values must be plain Python lists - use .tolist() on numpy/pandas objects

Histogram - always use pd.cut(), never hardcode bin edges:
```python
bins   = pd.cut(df["salary"], bins=8)
labels = bins.value_counts(sort=False).index.astype(str).tolist()
data   = bins.value_counts(sort=False).tolist()
```

scatter color_by - CRITICAL:
- color_by must use a column with FEWER THAN 10 unique values
- NEVER use: job_title, name, id, or any high-cardinality column → renders as single-colour blob
- Prefer: class/tier columns, boolean columns, or bins you create:
    color_by = pd.cut(df["salary"], bins=4, labels=["Q1","Q2","Q3","Q4"]).astype(str).tolist()
- CORRECT: df["job_survival_class"].astype(str).tolist()   ← 3 values
- WRONG:   df["job_title"].tolist()                        ← hundreds of values
